
# 基础框架
针对你想搭建的“通用无人机仿真实验框架”，我推荐采用**基于组件注册与动态组装的核心引擎 + Gym/类似Gym的强化学习接口**的设计。下面我从目标出发，帮你梳理出完整的架构、遗漏的模块、RL与LLM的集成方式，以及可测试、可插拔的实现思路。

---

### 1. 总体架构设计

整体分为四层：

- **核心模型层（Models）**：每个独立物理/逻辑模块都被抽象为接口，支持多实现、可单元测试。
- **环境组装层（Env Builder）**：通过配置字典或 YAML 文件，将模型实例组合成一个完整的 `gym.Env`。
- **强化学习接口层（Gym Adapter）**：严格遵循 `gym` 规范（`step`, `reset`, `render`），支持 `gymnasium>=0.26`，可直接接入 Stable-Baselines3、RLlib、CleanRL 等。
- **LLM 交互层（LLM Wrapper/Plugin）**：为环境注入自然语言语义，支持 LLM 作为策略部件或任务生成器。

### 2. 你已经列出的模块（以及细化建议）

你给出的五个模型已经构成了物理层和行为约束的核心，我会把它们抽象成统一的 **Component 接口**：

```python
class BaseComponent(ABC):
    @abstractmethod
    def reset(self, *args, **kwargs) -> None:
        """重置内部状态"""
    
    @abstractmethod
    def step(self, *args, **kwargs) -> Dict[str, Any]:
        """单步更新，返回该组件产生的影响量（如功耗、信噪比等）"""
```

#### 2.1 信道模型（Channel Model）
- 职责：计算无人机-无人机、无人机-地面站/边缘服务器的瞬时信噪比、数据速率。
- 可替换方案：
  - `FreeSpacePathLoss`：自由空间路径损耗
  - `ProbabilisticLOS`：概率视距/非视距（城区、郊区）
  - `RayleighRicianFading`：快衰落叠加
  - `DataDrivenChannel`：用数据集或神经网络拟合的信道
- 关键输出：`snr_db`, `achievable_rate` (bps), `outage_prob`

#### 2.2 动力模型（Mobility / Dynamics Model，注意：建议叫“移动模型/动力学模型”）
- 职责：根据动作（推力、角度变化）更新无人机位置、速度、加速度。
- 可替换方案：
  - `PointMass2D/3D`：简单质点模型
  - `QuadrotorDynamics`：四旋翼非线性动力学（含姿态）
  - `FixedWingKinematics`：固定翼运动学约束
  - `RandomWaypoint`：用于模拟非受控目标的随机移动
- 关键输出：新位置、速度、能耗（由功耗模型另算更清晰，但这里可输出推力等物理量）

#### 2.3 功耗模型（Power Model）
- 职责：根据飞行状态（悬停、机动）和计算负载，计算瞬时功率。
- 可替换方案：
  - `BasicHoverPower`：恒定悬停功率 + 行动功耗
  - `VelocityBasedModel`：功率与速度关系曲线
  - `ComputingPowerModel`：CPU/GPU 功耗与时钟频率/计算量的关系
- 关键输出：`propulsion_power`, `computation_power`, `total_power`

#### 2.4 状态模型（Observation Model）
- 职责：汇总当前可观测信息，输出给智能体的观测向量/字典。
- 可替换方案：
  - `GlobalObservation`：全状态可观测
  - `PartialObservable`：距离受限、遮挡导致的观测缺失
  - `MASObservation`：多智能体时各自的局部观测
  - `SemanticObservation`：为 LLM 生成文本状态描述
- 关键输出：`obs` (np.array 或 dict), `info`

#### 2.5 奖励模型（Reward Model）
- 职责：从状态、动作、任务完成情况计算标量奖励。
- 可替换方案：
  - `SparseCompletionReward`：任务完成后给固定值
  - `WeightedSumReward`：多个目标的加权和（时延、能耗、吞吐量）
  - `ConstrainedReward`：含约束惩罚项
- 关键输出：`reward` (float), `done`

### 3. **你遗漏的关键模块（强烈建议补充）**

在任务卸载场景中，缺少这些模块会让环境无法闭环：

#### 3.1 任务模型（Task Model）
- 职责：生成到达系统的计算任务，定义任务属性。
- 可替换方案：
  - `PoissonArrival`：泊松过程，任务大小/周期服从某种分布
  - `TraceDriven`：基于真实负载文件
  - `LLMGeneratedTasks`：用 LLM 生成多样化描述性任务（文本 + 参数）
- 每个任务包含：`task_id`, `data_size` (bits), `required_cycles` (CPU cycles), `max_delay` (s), `priority`, `task_description` (可选，自然语言)

#### 3.2 计算模型（Computation Model）
- 职责：确定任务在本地/卸载端执行所需的时延和能耗。
- 可替换方案：
  - `LocalCPU`：DVFS 模型，执行时间 = 所需周期数 / 分配的 CPU 频率
  - `EdgeServer`：固定或可变的边缘处理能力，可能引入排队
  - `U2UOffloading`：其他无人机充当移动边缘节点，结合信道传输
- 关键输出：`execution_time`, `energy_consumed`, `queue_length`

#### 3.3 能量模型（Energy Model）
- 职责：跟踪电池剩余电量，管理能量约束。
- 可替换方案：
  - `InfiniteBattery`：不关心能量
  - `FiniteBattery`：初始能量固定，消耗完即 crash
  - `EnergyHarvesting`：太阳能收集模型
- 关键输出：`remaining_energy`, `out_of_energy` flag

#### 3.4 卸载决策模型（Offloading Policy Interface）
- 职责：将 RL 智能体的**动作**映射为具体的卸载目标、资源分配、路径点等。
- 这不是环境要“实现策略”，而是定义**动作空间**和**动作解释规则**。
- 你需要一个 `ActionInterpreter` 组件：
  - 输入：原始动作（离散/连续）
  - 输出：`target_device`（本地/边缘1/边缘2），`offloading_ratio`，`computing_resource_allocation`，`thrust_vector`

#### 3.5 任务执行与调度模型（Scheduler / Queuing Model）
- 职责：管理多任务在本地和卸载端的并发执行，决定先处理哪个任务（FIFO、优先级等）。
- 如果忽略了调度，你就无法真正算出每项任务的实际完成时间，也就无法驱动奖励函数。

#### 3.6 信道接入与干扰模型（可选，但更真实）
- 职责：多个无人机共享频谱时的干扰、MAC 层竞争（TDMA/CSMA）。
- 输出：调整后的有效数据速率。

### 4. 完整的组装与环境生命周期

使用一个 **EnvBuilder** 来根据配置生成 Gym 环境：

```python
config = {
    "mobility": {"type": "QuadrotorDynamics", "params": {"dt": 0.1}},
    "channel":  {"type": "ProbabilisticLOS", "params": {"freq": 2.4e9}},
    "power":    {"type": "VelocityBasedModel", "params": {...}},
    "task":     {"type": "PoissonArrival", "params": {"lambda": 2}},
    "computing":{"type": "LocalCPU", "params": {"f_cpu": 1e9}},
    "energy":   {"type": "FiniteBattery", "params": {"capacity": 1000}},
    "reward":   {"type": "WeightedSumReward", "params": {"weights": [1,-0.1]}},
    "observation": {"type": "PartialObservable", "params": {"range": 500}},
    "action_interpreter": {"type": "DiscreteOffloading", "params": {"num_edges": 3}},
    ...
}
env = EnvBuilder.build(config)  # 返回 gym.Env
```

环境内部核心循环（在 `step` 中）：
1. `action_interpreter.process(action)` → 得到卸载目标和资源分配
2. `mobility.step(action.thrust)` → 更新位置
3. `channel.step(positions)` → 计算链路速率
4. `computing.step(tasks, offload_decision)` → 计算执行时延、能耗
5. `power.step(velocity, computing_load)` → 计算总能耗
6. `energy.step(total_power)` → 更新剩余电量
7. `task.step()` → 生成新任务，更新队列
8. `reward_model.calculate(state_info)` → 产生奖励
9. `observation_model.get_obs(state_info)` → 返回观测

### 5. 与 Gym 和常用 RL 库的对接

只要你的环境继承 `gym.Env`，实现了 `observation_space` 和 `action_space`，就能直接：

- 使用 **Stable-Baselines3**：`model = PPO('MultiInputPolicy', env)` 
- 使用 **RLlib**：`register_env("drone_env", lambda config: EnvBuilder.build(config))`
- 使用 **CleanRL** 的单文件脚本

建议采用 **Gymnasium** 新接口（`terminated`, `truncated`），并让环境返回 `info` 中包含详细指标，方便做实验分析。

### 6. LLM 如何结合进框架

LLM 在此类框架中通常不直接替换底层物理仿真，而是与高层决策、场景生成、人机交互相关。我建议提供两个扩展点：

#### 6.1 LLM 赋能的策略模块（插件形式）
- 将环境观测转换为自然语言字符串：`"无人机位于 (50, 100, 30)，电池 70%，当前有一个视频分析任务需要 1.2e12 CPU 周期，最大容忍时延 2 秒，附近有两个边缘节点。"`
- LLM 直接输出结构化动作：例如 `{"offload_to": "edge_1", "resource": 0.8}`，然后由 `ActionInterpreter` 解析。
- 这个 LLM-policy 可以作为 RL 的“教师模型”，生成示范数据；也可以直接作为可训练的 actor（RLHF / 微调）。

#### 6.2 任务/场景生成器
- 利用 LLM 生成丰富的、带自然语言描述的任务集合（比如“搜索区域A内的受伤人员视频传输”、“实时三维地图更新”）。
- 任务描述可以输入给智能体（若用了多模态状态），也可以单纯用来做场景多样性增强。

在框架中，我会设计一个 `LLMInterface` 适配器，它包裹环境并负责对话状态转换，而环境本身保持纯数值驱动，这样既能解耦又能自由切换“是否使用 LLM”。

### 7. 测试与验证设计

每个模块都有独立的单元测试（pytest），验证：
- 典型输入下输出范围是否合理
- 边界情况（速度为0时功耗不为0、信道距离无穷大时SNR趋近0）

集成测试：
- 随机策略跑 100 步不崩溃
- 观察输出维度是否符合 `observation_space`
- 能量耗尽后是否正确终止

这依赖于组件的“无副作用”设计：每个 `step()` 只依赖自己的历史状态和当前输入，不修改全局，方便构建 mock。

### 8. 有没有其他遗漏？

除了上述，还有几个锦上添花但可能重要的点：

- **环境噪声/干扰注入**：传感器噪声、定位误差、信道估计误差，可单独抽出一个 `NoiseModel` 注入到观测或信道输出中。
- **安全/避障模型**：如果需要物理避障，可加入 `CollisionModel`，检测无人机之间或与障碍物的碰撞，并作为惩罚或终止条件。
- **多智能体支持**：若有多架无人机协同，环境需支持 MultiAgent 接口（可参考 PettingZoo 或 RLLib 的多智能体规范），各模型要能处理多实体状态。
- **可视化与日志**：提供 `render()` 使用 matplotlib/plotly 或简单的 2D 图，便于调试；`Recorder` 组件保存 episode 数据到 csv / json，方便事后分析。

## 1. 核心抽象与注册器

### `core/base_component.py`
```python
from abc import ABC, abstractmethod

class BaseComponent(ABC):
    """所有可替换模块的根接口"""

    @abstractmethod
    def reset(self, **kwargs) -> None:
        """重置内部状态到初始条件"""
        pass

    def step(self, **kwargs):
        """单步更新，由子类按需实现"""
        raise NotImplementedError
```

### `core/interfaces.py`
定义了每个领域模块的接口，方便测试和替换。

```python
from abc import abstractmethod
from .base_component import BaseComponent

class IMobility(BaseComponent):
    @abstractmethod
    def step(self, action: 'np.ndarray') -> dict:
        """根据动作返回新的位置、速度等"""
        pass

class IChannel(BaseComponent):
    @abstractmethod
    def rate(self, tx_pos, rx_pos) -> float:
        """返回信道速率 bps"""
        pass

class IPower(BaseComponent):
    @abstractmethod
    def compute(self, velocity, computation_load) -> float:
        """返回总功耗"""
        pass

class ITaskGenerator(BaseComponent):
    @abstractmethod
    def sample(self, current_time) -> list:
        """返回当前时间步到达的任务列表"""
        pass

class IComputing(BaseComponent):
    @abstractmethod
    def process(self, task, offload_target, channel_rate) -> dict:
        """处理任务，返回执行时间、能耗、是否成功"""
        pass

class IEnergy(BaseComponent):
    @abstractmethod
    def consume(self, power, dt) -> None:
        """消耗能量"""
        pass
    @abstractmethod
    def remaining(self) -> float:
        """剩余能量"""
        pass

class IReward(BaseComponent):
    @abstractmethod
    def compute(self, state) -> float:
        pass

class IObservation(BaseComponent):
    @abstractmethod
    def get_obs(self, env_state) -> 'np.ndarray':
        pass

class IActionInterpreter(BaseComponent):
    @abstractmethod
    def interpret(self, raw_action) -> tuple:
        """将智能体原始动作映射为移动指令、卸载选择、资源分配"""
        pass
```

### `core/env_builder.py`
提供类型注册和从 YAML 构建环境的工厂。

```python
import yaml
from typing import Dict, Type

class EnvBuilder:
    _registry: Dict[str, Type] = {}

    @classmethod
    def register(cls, name: str, component_class: Type):
        cls._registry[name] = component_class

    @classmethod
    def build_from_yaml(cls, yaml_path: str):
        with open(yaml_path, 'r') as f:
            config = yaml.safe_load(f)
        return cls.build(config['modules'])

    @classmethod
    def build(cls, module_cfg: dict):
        # 实例化每个模块
        modules = {}
        for module_name, cfg in module_cfg.items():
            if module_name in ('env_params', 'meta'):  # 非模块配置跳过
                continue
            if isinstance(cfg, dict) and 'type' in cfg:
                modules[module_name] = cls._instantiate(cfg)
            # 否则可能是嵌套结构，留待后续扩展
        # 从 env_params 中提取环境参数
        env_params = module_cfg.get('env_params', {})
        from envs.drone_env import DroneEnv  # 避免循环依赖
        env = DroneEnv(
            mobility=modules['mobility'],
            channel=modules['channel'],
            power=modules['power'],
            task_gen=modules['task'],
            computing=modules['computing'],
            energy=modules['energy'],
            reward_model=modules['reward'],
            obs_model=modules['observation'],
            action_interp=modules['action_interpreter'],
            server_pos=env_params.get('server_pos', [300, 0, 50]),
            max_steps=env_params.get('max_steps', 200)
        )
        return env

    @classmethod
    def _instantiate(cls, cfg: dict):
        comp_type = cfg['type']
        params = cfg.get('params', {})
        if comp_type not in cls._registry:
            raise ValueError(f"Module type '{comp_type}' not registered.")
        return cls._registry[comp_type](**params)
```

## 2. 各模型的最小实现

每个模型文件都包含一个最简实现，用 `@EnvBuilder.register(...)` 装饰。

### `models/mobility.py`
```python
import numpy as np
from core.interfaces import IMobility
from core.env_builder import EnvBuilder

@EnvBuilder.register('SimplePointMass')
class SimplePointMass(IMobility):
    def __init__(self, dt=1.0, max_speed=10.0, bounds=500.0):
        self.dt = dt
        self.max_speed = max_speed
        self.bounds = bounds

    def reset(self, init_pos=None, **kwargs):
        self.pos = np.array(init_pos if init_pos else [0.,0.,50.])
        self.vel = np.zeros(3)

    def step(self, action):
        # action: 2D 加速度 [ax, ay]
        acc = np.clip(action, -2.0, 2.0)
        self.vel[:2] += acc * self.dt
        speed = np.linalg.norm(self.vel[:2])
        if speed > self.max_speed:
            self.vel[:2] = self.vel[:2] / speed * self.max_speed
        self.pos += self.vel * self.dt
        self.pos[:2] = np.clip(self.pos[:2], -self.bounds, self.bounds)
        return {'pos': self.pos.copy(), 'vel': self.vel.copy()}
```

### `models/channel.py`
```python
import numpy as np
from core.interfaces import IChannel
from core.env_builder import EnvBuilder

@EnvBuilder.register('FreeSpace')
class FreeSpaceChannel(IChannel):
    def __init__(self, freq=2.4e9, tx_power_dbm=20, bandwidth=10e6):
        self.freq = freq
        self.tx_power = 10**((tx_power_dbm-30)/10)
        self.B = bandwidth
        self.N0 = 1.38e-23 * 290 * bandwidth

    def reset(self, **kwargs): pass

    def rate(self, tx_pos, rx_pos):
        d = np.linalg.norm(np.array(tx_pos)-np.array(rx_pos)) + 1e-6
        path_loss = (4*np.pi*d*self.freq/3e8)**2
        rx_power = self.tx_power / path_loss
        snr = rx_power / self.N0
        return self.B * np.log2(1 + snr)
```

```python
import numpy as np
from core.interfaces import ITaskGenerator
from core.env_builder import EnvBuilder

@EnvBuilder.register('PoissonArrival')
class PoissonArrival(ITaskGenerator):
    def __init__(self, arrival_rate=0.3, data_size_mean=1e6, req_cycles_mean=5e8, max_delay=2.0):
        self.rate = arrival_rate
        self.data_mean = data_size_mean
        self.cycles_mean = req_cycles_mean
        self.max_delay = max_delay
        self.next_id = 0

    def reset(self, **kwargs):
        self.next_id = 0

    def sample(self, current_time):
        n = np.random.poisson(self.rate)
        tasks = []
        for _ in range(n):
            tasks.append({
                'id': self.next_id,
                'data_size': np.random.exponential(self.data_mean),
                'req_cycles': np.random.exponential(self.cycles_mean),
                'max_delay': self.max_delay,
                'arrival_time': current_time
            })
            self.next_id += 1
        return tasks
```

```python
from core.interfaces import IComputing
from core.env_builder import EnvBuilder

@EnvBuilder.register('EdgeOffloading')
class EdgeOffloading(IComputing):
    def __init__(self, edge_freq=3e9, local_freq=1e9):
        self.edge_freq = edge_freq
        self.local_freq = local_freq

    def reset(self, **kwargs): pass

    def process(self, task, offload_target, channel_rate):
        if offload_target == 0:  # 本地
            time = task['req_cycles'] / self.local_freq
            energy = time * 1e-11 * self.local_freq**2  # 简化能量模型
            return {'exec_time': time, 'energy': energy, 'success': True}
        else:  # 卸载到边缘
            if channel_rate <= 0:
                return {'exec_time': 1e6, 'energy': 0, 'success': False}
            tx_time = task['data_size'] / channel_rate
            proc_time = task['req_cycles'] / self.edge_freq + 0.02
            total_time = tx_time + proc_time
            comm_energy = tx_time * 2.0  # 2W 发射功率
            return {'exec_time': total_time, 'energy': comm_energy, 'success': True}
```

```python
import gymnasium as gym
from gymnasium import spaces
import numpy as np

class DroneEnv(gym.Env):
    metadata = {'render_modes': []}

    def __init__(self, mobility, channel, power, task_gen, computing, energy,
                 reward_model, obs_model, action_interp, server_pos, max_steps=200):
        super().__init__()
        self.mobility = mobility
        self.channel = channel
        self.power = power
        self.task_gen = task_gen
        self.computing = computing
        self.energy_model = energy
        self.reward_model = reward_model
        self.obs_model = obs_model
        self.action_interp = action_interp
        self.server_pos = np.array(server_pos)
        self.max_steps = max_steps

        self.action_space = spaces.Box(
            low=np.array([-1,-1,0,0.5]),
            high=np.array([1,1,1,2.0]),
            dtype=np.float32
        )
        self.observation_space = spaces.Box(
            low=-np.inf, high=np.inf, shape=(8,), dtype=np.float32
        )

        self.task_queue = []
        self.steps = 0
        self.total_completed = 0
        self.total_delay = 0.0
        self.total_energy = 0.0

    def reset(self, seed=None, options=None):
        super().reset(seed=seed)
        self.steps = 0
        self.task_queue = []
        self.total_completed = 0
        self.total_delay = 0.0
        self.total_energy = 0.0
        self.mobility.reset()
        self.energy_model.reset()
        self.task_gen.reset()
        return self._get_obs(), {}

    def step(self, action):
        move_cmd, offload_target, cpu_freq = self.action_interp.interpret(action)
        # 移动
        state = self.mobility.step(move_cmd)
        pos, vel = state['pos'], state['vel']
        # 信道
        rate = self.channel.rate(pos, self.server_pos)
        # 任务到达
        new_tasks = self.task_gen.sample(self.steps)
        self.task_queue.extend(new_tasks)
        # 处理一个任务（简化）
        delay, energy_cost = 0.0, 0.0
        if self.task_queue:
            task = self.task_queue.pop(0)
            result = self.computing.process(task, offload_target, rate)
            if result['success'] and result['exec_time'] <= task['max_delay']:
                self.total_completed += 1
                delay = result['exec_time']
            else:
                delay = task['max_delay']  # 超时视为浪费该任务
            energy_cost = result['energy']
        # 功耗
        power_total = self.power.compute(vel, cpu_freq if self.task_queue else 0)
        self.energy_model.consume(power_total, dt=1.0)
        self.total_delay += delay
        self.total_energy += power_total
        # 奖励
        reward = self.reward_model.compute({
            'completed': 1 if delay > 0 else 0,
            'energy_cost': power_total,
            'delay': delay
        })
        self.steps += 1
        terminated = self.energy_model.remaining() <= 0
        truncated = self.steps >= self.max_steps
        return self._get_obs(), reward, terminated, truncated, {
            'completed_tasks': self.total_completed,
            'remaining_energy': self.energy_model.remaining()
        }

    def _get_obs(self):
        state = self.mobility.step([0,0])  # 为了取当前状态，更优雅应缓存
        self.mobility.pos = state['pos']
        env_state = {
            'pos': state['pos'],
            'vel': state['vel'],
            'energy': self.energy_model.remaining(),
            'queue_len': len(self.task_queue),
            'server_pos': self.server_pos
        }
        return self.obs_model.get_obs(env_state)
```

## 2. 指标收集器 `evaluation/metrics.py`

该类在环境交互过程中自动捕获每一步的详细数据，每个 episode 结束后计算汇总指标。

```python
import numpy as np
from collections import defaultdict
from typing import List, Dict, Any

class MetricsTracker:
    """记录整个评估过程中的每个episode的详细和汇总指标"""
    def __init__(self):
        self.reset_session()

    def reset_session(self):
        self.episode_data = []          # 每个episode的完整数据字典
        self.current_episode = None     # 当前episode的逐步数据

    def begin_episode(self):
        self.current_episode = {
            'steps': [],
            'reward': [],
            'action': [],
            'task_completed': 0,
            'task_failed': 0,
            'delay_list': [],
            'energy_consumed': 0.0,
            'offload_actions': [],  # 0: local, 1: edge
            'distance_list': [],
            'final_energy': 0.0
        }

    def record_step(self, obs, action, reward, info):
        if self.current_episode is None:
            return
        self.current_episode['steps'].append(len(self.current_episode['steps']))
        self.current_episode['reward'].append(reward)
        self.current_episode['action'].append(action)
        if 'offload' in info:
            self.current_episode['offload_actions'].append(info['offload'])
        if 'delay' in info:
            self.current_episode['delay_list'].append(info['delay'])
        if 'completed' in info and info['completed']:
            self.current_episode['task_completed'] += 1
        if 'failed' in info and info['failed']:
            self.current_episode['task_failed'] += 1
        if 'distance' in info:
            self.current_episode['distance_list'].append(info['distance'])

    def end_episode(self, final_info):
        """在episode结束后调用，汇总数据"""
        if self.current_episode is None:
            return
        ep = self.current_episode
        ep['total_reward'] = np.sum(ep['reward'])
        ep['avg_delay'] = np.mean(ep['delay_list']) if ep['delay_list'] else 0.0
        ep['energy_consumed'] = final_info.get('energy_consumed', 0.0)
        ep['offload_rate'] = np.mean([1 if a else 0 for a in ep['offload_actions']]) if ep['offload_actions'] else 0.0
        ep['avg_distance'] = np.mean(ep['distance_list']) if ep['distance_list'] else 0.0
        ep['completion_rate'] = ep['task_completed'] / (ep['task_completed'] + ep['task_failed']) if (ep['task_completed']+ep['task_failed'])>0 else 0.0
        self.episode_data.append(ep)
        self.current_episode = None

    def get_summary(self) -> Dict[str, Any]:
        """计算所有episode的均值和标准差"""
        if not self.episode_data:
            return {}
        rewards = [ep['total_reward'] for ep in self.episode_data]
        completions = [ep['task_completed'] for ep in self.episode_data]
        delays = [ep['avg_delay'] for ep in self.episode_data]
        energies = [ep['energy_consumed'] for ep in self.episode_data]
        offloads = [ep['offload_rate'] for ep in self.episode_data]
        distances = [ep['avg_distance'] for ep in self.episode_data]
        return {
            'num_episodes': len(self.episode_data),
            'avg_reward': (np.mean(rewards), np.std(rewards)),
            'avg_completed_tasks': (np.mean(completions), np.std(completions)),
            'avg_delay': (np.mean(delays), np.std(delays)),
            'avg_energy': (np.mean(energies), np.std(energies)),
            'avg_offload_rate': (np.mean(offloads), np.std(offloads)),
            'avg_distance': (np.mean(distances), np.std(distances)),
        }

    def to_pandas(self):
        """转换为DataFrame便于画图"""
        import pandas as pd
        rows = []
        for i, ep in enumerate(self.episode_data):
            rows.append({
                'episode': i,
                'reward': ep['total_reward'],
                'completed_tasks': ep['task_completed'],
                'avg_delay': ep['avg_delay'],
                'energy': ep['energy_consumed'],
                'offload_rate': ep['offload_rate'],
                'avg_distance': ep['avg_distance'],
            })
        return pd.DataFrame(rows)
```

## 3. 策略评估器 `evaluation/evaluator.py`

提供通用的评估函数，可接受任何能够产生动作的 predictor（RL模型或随机策略等）。

```python
import numpy as np
from .metrics import MetricsTracker
from typing import Callable, Dict, Any

def evaluate_policy(env, policy_fn: Callable, num_episodes: int = 10,
                    render: bool = False, verbose: bool = True) -> Dict[str, Any]:
    """
    评估一个策略并返回汇总指标。
    policy_fn(obs) -> action
    """
    tracker = MetricsTracker()
    for ep_idx in range(num_episodes):
        obs, _ = env.reset()
        tracker.begin_episode()
        done = False
        ep_step = 0
        while not done:
            action = policy_fn(obs)
            next_obs, reward, terminated, truncated, info = env.step(action)
            # 从info中提取所需字段（这里简化，需要env提供这些info）
            step_info = {
                'completed': info.get('completed', False),
                'failed': info.get('failed', False),
                'delay': info.get('delay', 0.0),
                'offload': info.get('offload_target', 0),
                'distance': info.get('distance', 0.0)
            }
            tracker.record_step(obs, action, reward, step_info)
            obs = next_obs
            done = terminated or truncated
            ep_step += 1
        # 收集最终信息
        final_info = {
            'energy_consumed': info.get('total_energy', 0.0)
        }
        tracker.end_episode(final_info)
        if verbose:
            print(f"Episode {ep_idx+1}/{num_episodes}, Reward: {tracker.episode_data[-1]['total_reward']:.2f}, "
                  f"Completed: {tracker.episode_data[-1]['task_completed']}")
    return tracker.get_summary(), tracker
```

- 可选对比不同配置（如仅本地执行 vs 边缘卸载）

```python
import os
import numpy as np
import yaml
import matplotlib.pyplot as plt
from stable_baselines3 import PPO
from stable_baselines3.common.callbacks import BaseCallback

# 导入框架
from core.env_builder import EnvBuilder
from evaluation.evaluator import evaluate_policy
from evaluation.metrics import MetricsTracker
import models.mobility, models.channel, models.power, models.task, models.computing, models.energy, models.reward, models.observation, models.action_interpreter

class TrainingLogger(BaseCallback):
    def __init__(self, verbose=0):
        super().__init__(verbose)
        self.episode_rewards = []
        self.current_ep_reward = 0.0
        self.timesteps = []

    def _on_step(self):
        self.current_ep_reward += self.locals['rewards'][0]
        if self.locals['dones'][0]:
            self.episode_rewards.append(self.current_ep_reward)
            self.timesteps.append(self.num_timesteps)
            self.current_ep_reward = 0.0
        return True

def train_ppo(env, total_timesteps=50000):
    model = PPO('MlpPolicy', env, verbose=0)
    logger = TrainingLogger()
    model.learn(total_timesteps=total_timesteps, callback=logger)
    return model, logger

def plot_curve(logger, save_path='learning_curve.png'):
    plt.figure()
    plt.plot(logger.timesteps, logger.episode_rewards)
    plt.xlabel('Timesteps')
    plt.ylabel('Episode Reward')
    plt.title('PPO Training on Drone Offloading')
    plt.grid(True)
    plt.savefig(save_path)
    plt.close()

def main():
    # 主实验：边缘卸载
    print("=== Training with Edge Offloading ===")
    env_train = EnvBuilder.build_from_yaml('configs/default.yaml')
    model, logger = train_ppo(env_train, total_timesteps=50000)
    plot_curve(logger, 'learning_curve_edge.png')

    # 评估
    env_eval = EnvBuilder.build_from_yaml('configs/default.yaml')
    def policy_fn(obs):
        return model.predict(obs, deterministic=True)[0]

    summary, tracker = evaluate_policy(env_eval, policy_fn, num_episodes=20)
    print("Edge Offloading Evaluation Summary:")
    for k, v in summary.items():
        if isinstance(v, tuple):
            print(f"  {k}: {v[0]:.3f} +/- {v[1]:.3f}")
        else:
            print(f"  {k}: {v}")

    # 对比实验：仅本地计算（通过加载另一个 YAML）
    print("\n=== Training with Local Only ===")
    env_local = EnvBuilder.build_from_yaml('configs/compare_local.yaml')
    model_local, logger_local = train_ppo(env_local, total_timesteps=50000)
    plot_curve(logger_local, 'learning_curve_local.png')

    env_eval_local = EnvBuilder.build_from_yaml('configs/compare_local.yaml')
    def policy_local(obs):
        return model_local.predict(obs, deterministic=True)[0]
    summary_local, _ = evaluate_policy(env_eval_local, policy_local, num_episodes=20)
    print("Local Only Evaluation Summary:")
    for k, v in summary_local.items():
        if isinstance(v, tuple):
            print(f"  {k}: {v[0]:.3f} +/- {v[1]:.3f}")
        else:
            print(f"  {k}: {v}")

    # 绘制对比柱状图
    categories = ['Avg Reward', 'Completed Tasks', 'Avg Delay (s)', 'Avg Energy (J)', 'Offload Rate']
    edge_means = [summary['avg_reward'][0], summary['avg_completed_tasks'][0],
                  summary['avg_delay'][0], summary['avg_energy'][0], summary['avg_offload_rate'][0]]
    local_means = [summary_local['avg_reward'][0], summary_local['avg_completed_tasks'][0],
                   summary_local['avg_delay'][0], summary_local['avg_energy'][0], summary_local['avg_offload_rate'][0]]

    x = np.arange(len(categories))
    width = 0.35
    fig, ax = plt.subplots()
    ax.bar(x - width/2, edge_means, width, label='Edge Offloading')
    ax.bar(x + width/2, local_means, width, label='Local Only')
    ax.set_ylabel('Score')
    ax.set_title('Performance Comparison')
    ax.set_xticks(x)
    ax.set_xticklabels(categories)
    ax.legend()
    plt.xticks(rotation=45)
    plt.tight_layout()
    plt.savefig('comparison.png')
    plt.close()
    print("\nPlots saved: learning_curve_edge.png, learning_curve_local.png, comparison.png")

if __name__ == '__main__':
    main()
```

**对比实验的配置文件 `configs/compare_local.yaml`** 只需将 `computing.type` 改为 `LocalOnly`（需要先实现一个 `LocalOnly` 类，很简单，可参考前面的最小框架）。

## 8. 论文级实验的进一步扩展建议

你可以轻松在此框架上扩展更多论文需要的功能：

1. **多算法对比**：在 `train_and_eval.py` 中增加对 DQN、A2C、SAC 的训练和评估函数，使用相同的 `evaluate_policy`。
2. **多次运行与统计**：用不同随机种子重复训练（通过设置 `env.seed` 和模型 seed），收集多次运行的均值和方差，并在图中绘制误差带。
3. **时延分布图**：`MetricsTracker` 已记录每个任务的时延，可以绘制 CDF 或箱线图（使用 `tracker.to_pandas()`）。
4. **消融实验**：通过修改 YAML 切换模块（如去掉能量约束、改变信道模型），复用评估管道。
5. **表格生成**：将 `summary` 字典直接导出为 csv 或 LaTeX 表格。
6. **LLM 策略评估**：若你实现了 `LLMActionInterpreter`，只需将 `policy_fn` 换成 LLM 调用的包装函数，即可进行对比。

所有这些都不需要改动框架核心，只需在 `evaluation/` 和 `scripts/` 中添加新脚本即可。

这套评估体系与之前的模块化设计无缝衔接，既满足了训练需求，也补足了论文所需的系统化实验与指标产出。你可以直接在上面构建更复杂的实验方案，并得到规范的图表和数据。

# 补充分离式脚本

你说得很对，实验流程中的训练、评估、画图应该是**独立的操作**，这样你可以：
- 用不同超参多次训练，保存模型，不用每次重新训练。
- 对不同模型统一评估，生成可对比的指标。
- 灵活改变可视化样式而不重新跑环境。

我帮你将之前的 `train_and_eval.py` 拆分为三个独立脚本，同时提供一个可选的一体化脚本。它们共享底层的 `MetricsTracker` 和 `evaluate_policy`，保证指标一致性。

## 2. 独立训练脚本 `scripts/train.py`

```python
import argparse
import os
import json
import numpy as np
from stable_baselines3 import PPO
from stable_baselines3.common.callbacks import BaseCallback
from core.env_builder import EnvBuilder
# 导入所有模型模块以触发注册
import models.mobility, models.channel, models.power, models.task, models.computing
import models.energy, models.reward, models.observation, models.action_interpreter

class TrainingLogger(BaseCallback):
    def __init__(self, verbose=0):
        super().__init__(verbose)
        self.ep_rewards = []
        self.timesteps = []
        self.ep_reward_buf = 0.0

    def _on_step(self):
        self.ep_reward_buf += self.locals['rewards'][0]
        if self.locals['dones'][0]:
            self.ep_rewards.append(float(self.ep_reward_buf))
            self.timesteps.append(int(self.num_timesteps))
            self.ep_reward_buf = 0.0
        return True

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--config', type=str, default='configs/default.yaml',
                       help='Path to YAML config file')
    parser.add_argument('--total_timesteps', type=int, default=50000)
    parser.add_argument('--save_model', type=str, default='trained_models/ppo_drone',
                       help='Path prefix for saving model')
    parser.add_argument('--log_dir', type=str, default='logs/')
    args = parser.parse_args()

    os.makedirs('trained_models', exist_ok=True)
    os.makedirs(args.log_dir, exist_ok=True)

    env = EnvBuilder.build_from_yaml(args.config)
    model = PPO('MlpPolicy', env, verbose=1)
    logger = TrainingLogger()
    model.learn(total_timesteps=args.total_timesteps, callback=logger)
    
    # 保存模型
    model.save(args.save_model)
    
    # 保存训练曲线数据
    log_data = {'timesteps': logger.timesteps, 'episode_rewards': logger.ep_rewards}
    csv_path = os.path.join(args.log_dir, 'training_log.json')
    with open(csv_path, 'w') as f:
        json.dump(log_data, f)
    
    print(f"Model saved to {args.save_model}.zip")
    print(f"Training log saved to {csv_path}")

if __name__ == '__main__':
    main()
```

## 3. 独立评估脚本 `scripts/eval.py`

```python
import argparse
import json
import os
import numpy as np
from stable_baselines3 import PPO
from core.env_builder import EnvBuilder
from evaluation.evaluator import evaluate_policy
import models.mobility, models.channel, models.power, models.task, models.computing
import models.energy, models.reward, models.observation, models.action_interpreter

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--config', type=str, default='configs/default.yaml',
                       help='YAML config for evaluation environment')
    parser.add_argument('--model', type=str, required=True,
                       help='Path to trained model (without .zip)')
    parser.add_argument('--episodes', type=int, default=20)
    parser.add_argument('--output', type=str, default='eval_metrics.json')
    parser.add_argument('--seed', type=int, default=42)
    args = parser.parse_args()

    env = EnvBuilder.build_from_yaml(args.config)
    # 如果需要在评估时固定随机种子
    env.reset(seed=args.seed)
    
    model = PPO.load(args.model)
    def policy_fn(obs):
        return model.predict(obs, deterministic=True)[0]

    summary, tracker = evaluate_policy(env, policy_fn, num_episodes=args.episodes)
    
    # 转换 summary 中的 tuple 为 list 以支持 JSON 序列化
    summary_serializable = {}
    for k, v in summary.items():
        if isinstance(v, tuple):
            summary_serializable[k] = {'mean': v[0], 'std': v[1]}
        else:
            summary_serializable[k] = v
    
    with open(args.output, 'w') as f:
        json.dump(summary_serializable, f, indent=2)
    
    print("Evaluation Summary:")
    for k, v in summary.items():
        if isinstance(v, tuple):
            print(f"  {k}: {v[0]:.4f} +/- {v[1]:.4f}")
        else:
            print(f"  {k}: {v}")

if __name__ == '__main__':
    main()
```

## 4. 独立画图脚本 `scripts/plot.py`

```python
import argparse
import json
import matplotlib.pyplot as plt
import numpy as np
import os

def plot_learning_curve(log_path, save_path='learning_curve.png', window=5):
    with open(log_path, 'r') as f:
        data = json.load(f)
    timesteps = data['timesteps']
    rewards = data['episode_rewards']
    
    # 滑动平均
    smoothed = np.convolve(rewards, np.ones(window)/window, mode='valid')
    ts_smoothed = timesteps[window-1:]
    
    plt.figure()
    plt.plot(ts_smoothed, smoothed)
    plt.xlabel('Timesteps')
    plt.ylabel('Episode Reward (smoothed)')
    plt.title('Training Learning Curve')
    plt.savefig(save_path)
    print(f"Learning curve saved to {save_path}")

def plot_evaluation_summary(json_path, save_path='eval_summary.png'):
    with open(json_path, 'r') as f:
        summary = json.load(f)
    
    metrics = ['avg_reward', 'avg_completed_tasks', 'avg_delay', 'avg_energy', 'avg_offload_rate']
    labels = ['Avg Reward', 'Completed Tasks', 'Avg Delay (s)', 'Avg Energy (J)', 'Offload Rate']
    
    means = [summary[m]['mean'] for m in metrics if m in summary]
    stds = [summary[m]['std'] for m in metrics if m in summary]
    
    x = np.arange(len(means))
    plt.figure(figsize=(10, 6))
    plt.bar(x, means, yerr=stds, capsize=5, color='skyblue', edgecolor='black')
    plt.xticks(x, labels, rotation=45)
    plt.ylabel('Value')
    plt.title('Evaluation Metrics')
    plt.tight_layout()
    plt.savefig(save_path)
    print(f"Evaluation summary plot saved to {save_path}")

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--type', type=str, required=True, choices=['train_curve', 'eval_summary'],
                       help='Type of plot to generate')
    parser.add_argument('--input', type=str, required=True,
                       help='Path to training log JSON or eval metrics JSON')
    parser.add_argument('--output', type=str, default='plot.png',
                       help='Output image path')
    parser.add_argument('--window', type=int, default=5,
                       help='Smoothing window for learning curve')
    args = parser.parse_args()
    
    if args.type == 'train_curve':
        plot_learning_curve(args.input, args.output, args.window)
    elif args.type == 'eval_summary':
        plot_evaluation_summary(args.input, args.output)

if __name__ == '__main__':
    main()
```

## 7. 进一步组合实验的批次脚本

你还可以编写一个简单的 Shell 脚本或 Python 脚本，批量运行多种配置的“训练+评估+画图”：

```bash
#!/bin/bash
configs=("configs/default.yaml" "configs/compare_local.yaml")
models=("trained_models/edge" "trained_models/local")

for i in "${!configs[@]}"; do
    echo "Training ${models[$i]}"
    python scripts/train.py --config "${configs[$i]}" --save_model "${models[$i]}" --total_timesteps 50000
    python scripts/eval.py --config "${configs[$i]}" --model "${models[$i]}" --output "results/${models[$i]##*/}_eval.json"
done
python scripts/plot.py --type eval_summary --input results/edge_eval.json --output plots/edge.png
python scripts/plot.py --type eval_summary --input results/local_eval.json --output plots/local.png
```

---

通过这种拆分，你获得了极大的灵活性：
- **多策略对比**：只需用不同模型调用 `eval.py`，输出各自 JSON，然后可自行编写对比绘图脚本。
- **调试**：训练一次，评估 N 次，调整评估种子或环境参数，而不必重新训练。
- **论文复现**：可将训练日志和评估指标归档，所有图表均能从 JSON 数据重新生成，满足可重复性要求。

原有的框架结构和注册机制完全不变，你只需在 `scripts/` 中添加这几个文件即可。这样的设计既满足了你的独立操作需求，又保留了快速验证的一体化接口。
